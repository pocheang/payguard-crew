## 🔄 Documentation Standards

### File Naming Convention
- UPPERCASE_WITH_UNDERSCORES.md for root-level docs
- lowercase_with_underscores.md for docs/ subdirectory
- Use descriptive, self-explanatory names

### Document Structure
All documentation follows this structure:
1. **Title** - Clear, descriptive heading
2. **Overview** - Brief introduction
3. **Content** - Organized sections with headers
4. **Examples** - Code samples where applicable
5. **Metadata** - Last updated date, version

### Markdown Standards
- Use ATX-style headers (# ## ###)
- Code blocks with language specification
- Tables for structured data
- Links for cross-references
- Emojis for visual markers (✅ ⚠️ 📊)
